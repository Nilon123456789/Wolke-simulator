Le pas de temps définit l'intervalle de temps entre deux itérations. Plus le pas de temps est petit, plus la simulation est précise. Cependant, un pas de temps petit augmente le temps de calcul. Si le pas de temps est trop élevé, la simulation risque fortement de devenir instable. Si c'est le cas dans la console le moteur physique affiche une value CFL qui corespond a la stabilité de la simulation. La simulation est considéré comme stable tant que la valeur est inférieur ou égale à 1. De plus, dans le message du moteur physique avec le CFL une sugestion de pas de temps est suggéré.
